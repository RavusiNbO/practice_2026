Проект запускает MCP-сервер, PostgreSQL, Qdrant, Loki и Grafana в Docker Compose.
PDF-документы из `server/documents` индексируются в Qdrant и доступны серверу для семантического поиска.

## Требования

- Docker Desktop с Docker Compose;
- GNU Make.

В `compose.yaml` явно указана платформа `linux/amd64`. Контейнеры будут работать через эмуляцию и могут запускаться медленнее, если запускать на других платформах.

## Быстрый старт

Из корня репозитория выполните:

```bash
make up
make init-loki
make index
```

`make index` удаляет текущую коллекцию `documents` в Qdrant и создаёт её заново на основе всех PDF из `server/documents`. Индексация может занять некоторое время.

Проверить состояние и открыть поток логов:

```bash
make ps
make logs
```

Остановить проект:

```bash
make down
```

## Добавить PDF

Передайте абсолютный или относительный путь к файлу:

```bash
make add-pdf PDF="/Users/name/Downloads/manual.pdf"
make index
```

Команда копирует файл в `server/documents`. Она не перезаписывает PDF с тем же именем; переименуйте исходный файл или удалите существующий вручную, если хотите заменить его.

После добавления нужно выполнить `make index`, поскольку индексатор всегда переиндексирует весь каталог документов.

## Инструмент для проверки MCP
`npx @modelcontextprotocol/inspector --no-oauth`

Дальше запустите контейнеры, подключитесь к localhost:8000 и сможете тестировать tools

## Все команды

```bash
make help
```

| Команда | Назначение |
| --- | --- |
| `make up` | Собрать образы и запустить сервисы в фоне. |
| `make down` | Остановить и удалить контейнеры. Данные именованных Docker-томов сохраняются. |
| `make build` | Пересобрать образы. |
| `make restart` | Перезапустить сервисы. |
| `make ps` | Показать состояние сервисов. |
| `make logs` | Смотреть логи всех сервисов. |
| `make init-loki` | Настроить отправку логов MCP-сервиса в Loki. |
| `make index` | Переиндексировать PDF из `server/documents`. |
| `make add-pdf PDF="..."` | Скопировать PDF в каталог документов. |
